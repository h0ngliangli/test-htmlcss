JavaScript 中 undefined 的用法

在 JavaScript 中，undefined 是一个基本数据类型，用来表示一个变量已经声明但尚未赋值，或者对象的某个属性不存在。以下是 undefined 的几种常见用法和场景：

1. 变量未赋值时的 undefined

当一个变量声明后未赋值，它的默认值就是 undefined。

let a;
console.log(a); // 输出: undefined

2. 使用 typeof 检测是否为 undefined

可以使用 typeof 操作符来检查一个变量是否为 undefined。这种方法比较安全，即使变量未声明也不会报错。 

let b;
console.log(typeof b === 'undefined'); // 输出: true

如果变量未声明，typeof 同样不会抛出错误，而是返回 'undefined'： 

console.log(typeof c === 'undefined'); // 输出: true

3. 函数没有返回值时返回 undefined

在 JavaScript 中，如果一个函数没有显式地使用 return 语句返回值，那么该函数默认返回 undefined。 

function myFunction() {}
console.log(myFunction()); // 输出: undefined

4. 对象属性不存在时的 undefined

当你访问一个对象中不存在的属性时，会返回 undefined。

let obj = { name: 'Alice' };
console.log(obj.age); // 输出: undefined

5. 数组中不存在的索引返回 undefined

访问数组中不存在的元素，也会返回 undefined。

let arr = [1, 2, 3];
console.log(arr[5]); // 输出: undefined

6. 函数参数未传递时为 undefined

在调用函数时，如果某个参数没有传递，那么该参数的值为 undefined。 

function greet(name) {
  console.log(name); // 如果未传入 name，输出: undefined
}

greet(); // 输出: undefined

7. 显式赋值 undefined

你可以显式地将变量或对象属性赋值为 undefined，虽然通常不推荐这样做，因为 null 更适合表示 "空值"。 

let x = undefined;
console.log(x); // 输出: undefined

8. 判断是否为 undefined

可以通过严格等于（===）判断某个变量是否为 undefined。 

let y;
if (y === undefined) {
  console.log('y 是 undefined');
}

9. 默认参数

当函数的某些参数可能没有传递时，可以设置默认值来避免 undefined。 

function sayHello(name = 'Guest') {
  console.log(`Hello, ${name}!`);
}

sayHello(); // 输出: Hello, Guest!

小结

undefined 主要用于表示变量或属性未赋值的状态，它与 null 是不同的概念，null 通常用于表示空值或不存在的对象，而 undefined 表示“未定义”或“缺失”。
