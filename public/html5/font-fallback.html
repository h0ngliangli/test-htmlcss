Font Fallback

Arial 是一种广泛使用的西文字体，它本身并不包含中文字符。然而，当你在网页的 font-family 中指定了 Arial，并遇到需要显示中文字符时，浏览器会使用一种机制称为 字体回退机制（font fallback） 来显示这些字符。 字体回退机制（Font Fallback）

浏览器在渲染文本时，如果指定的字体（如 Arial）不支持某些字符（比如中文），它会自动回退到系统中支持这些字符的字体来显示。这个过程对用户是透明的，浏览器会找到合适的字体来显示那些未包含在指定字体中的字符。 具体过程：

  • 当你指定 font-family: Arial 时，浏览器会优先尝试使用 Arial 来渲染网页内容。
  • 对于西文字母（A-Z、a-z）或其他 Arial 支持的字符，它会使用 Arial 字体渲染。
  • 当遇到 Arial 不支持的字符（如中文），浏览器会寻找系统中安装的默认中文字体来显示这些字符。 在 Windows 系统上，可能会回退到 "Microsoft YaHei"（微软雅黑） 或 "SimSun"（宋体）。 在 macOS 系统上，可能会回退到 "PingFang"、"Hiragino Sans" 等支持中文的字体。

为什么中文网页仍然指定 Arial？

许多网页（包括中文网页）在设置 font-family 时会使用一种“字体堆栈（font stack）”的策略，即指定多个字体，浏览器会从左到右依次查找支持的字体。这样可以确保页面在不同系统和语言环境下都能正常显示，并提供一致的体验。 示例： 

    
body {
    font-family: Arial, "Microsoft YaHei", "PingFang SC", sans-serif;
}

在这个例子中，网页首先尝试使用 Arial 字体。如果系统中没有 Arial 或者 Arial 不支持某些字符（比如中文），则会回退到 "Microsoft YaHei" 或 "PingFang SC" 这样的中文字体。 即使没有明确指定中文字体，浏览器依然会根据系统的默认设置选择合适的中文字体进行显示。因此，你在一些中文网站上看到的文本，即使指定了 Arial，实际上显示中文字符的字体并不是 Arial，而是通过字体回退机制使用的系统默认中文字体。

总结：

Arial 本身不支持中文。当指定 Arial 时，浏览器会通过字体回退机制自动选择系统中支持中文的字体来显示中文字符。 网页中的 font-family 设置可以包含多个字体，确保不同语言和字符集都能正确显示。 常见中文字体包括 "Microsoft YaHei"（微软雅黑）、"SimSun"（宋体）、"PingFang" 等。 因此，中文网页设置 Arial 是为了兼容西文字符，但浏览器在遇到中文时会自动回退到适合的中文字体来渲染。
