自定义属性

使b用自定义属性的规范方法：data-* 属性 HTML5 规定使用 data-* 的形式来定义自定义属性，这样可以确保它们是合法的，并且不会影响 HTML 的解析或与标准属性发生冲突。 示例： html 


  <div data-shadow="true">This div has a custom data attribute</div>

如何通过 JavaScript 访问 data-* 属性： 你可以通过 JavaScript 来访问自定义的 data-* 属性。使用 dataset 对象可以轻松获取这些属性。 示例： html 


<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Custom Data Attribute Example</title>
  </head>
  <body>
    <div id="myDiv" data-shadow="true">
      This div has a custom data attribute
    </div>

    <script>
      const myDiv = document.getElementById("myDiv")
      // 通过 dataset 获取自定义属性的值
      const shadowValue = myDiv.dataset.shadow
      console.log(shadowValue) // 输出: "true"
    </script>
  </body>
</html>

解释： data-shadow="true" 是一个自定义的数据属性，表示该div 元素是否有某种阴影效果（true 表示有）。 在 JavaScript 中，使用 myDiv.dataset.shadow 来获取 data-shadow 的值，dataset 会将 data-* 属性转换为对象的属性。 为什么要使用 data-* 前缀？ 标准化：HTML5 提倡使用 data-* 形式来定义自定义属性。这样可以确保自定义属性与标准 HTML 属性不冲突。 兼容性：浏览器不会对 data-* 属性做特殊处理，但它们可以通过 JavaScript 方便地访问。这种属性对搜索引擎爬虫、表单提交等也不会有不良影响。 可读性与语义性：通过 data-* 前缀，开发者可以清楚地知道哪些属性是自定义的，不会与原生 HTML 属性混淆。 非标准属性（无 data- 前缀） 尽管你可以直接添加任何自定义属性，例如 


  <div shadow="true">

，但这并不是标准做法，可能会导致潜在的兼容性问题或与未来的 HTML 标准发生冲突。HTML5 规范并没有阻止你添加非标准的属性，但为了避免不必要的麻烦，建议使用 data-* 形式来添加自定义属性。 示例： html 


<div shadow="true">Non-standard custom attribute</div>

虽然这段代码不会立即引起问题，但如果 HTML 标准未来定义了 shadow 作为一个合法属性，它可能会与该属性冲突。 小结： 你可以在 HTML 元素中使用自定义属性，但推荐使用 data-* 前缀的方式。 使用 data-* 属性可以确保自定义属性与标准属性的兼容性，且可以通过 JavaScript 的 dataset 轻松访问这些属性的值。
